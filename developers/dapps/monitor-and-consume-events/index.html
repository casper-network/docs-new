Monitoring and Consuming Events

The Casper Event Sidecar is an application running alongside the node process, allowing subscribers to monitor the event stream without querying the node, thus receiving faster responses and reducing the load on the node. Users needing access to the JSON-RPC will still need to query the node directly.

An alternate name for this application is the SSE Sidecar because it uses the node's Event Stream API returning Server-Sent Events (SSEs) in JSON format. The SSE Sidecar uses the node's Event Stream API to achieve the following goals:

  • Build a middleware service that connects to the Node Event Stream, replicating the SSE interface and its filters (i.e., /main, /deploys, and /sigs with support for the use of the ?start_from= query to allow clients to get previously sent events from the Sidecar's buffer.)
  • Provide a new RESTful endpoint that is discoverable on the network.

Visit GitHub for the latest source code and information on system architecture.

The Event Stream​

The Sidecar's event stream endpoint is a passthrough for all the events emitted by the node to which the Sidecar connects. The Node's Event Stream page explains the various event types emitted by the node and available through the Sidecar service. Also, the section on Monitoring the Event Stream gives a brief example on how to monitor Sidecar events.

Faults by Public Key​

Retrieve the faults associated with a validator's public key. The path URL is <HOST:PORT>/faults/<public-key>. Enter a valid hexadecimal representation of a validator's public key.

Example:

curl -s http://127.0.0.1:18888/faults/01a601840126a0363a6048bfcbb0492ab5a313a1a19dc4c695650d8f3b51302703

Faults by Era​

Return the faults associated with an era, given a valid era identifier. The path URL is: <HOST:PORT>/faults/<era-ID>. Enter an era identifier.

Example:

curl -s http://127.0.0.1:18888/faults/2304

Finality Signatures by Block​

Retrieve the finality signatures in a block, given its block hash. The path URL is: <HOST:PORT>/signatures/<block-hash>. Enter a valid block hash.

Example:

curl -s http://127.0.0.1:18888/signatures/85aa2a939bc3a4afc6d953c965bab333bb5e53185b96bb07b52c295164046da2

Step by Era​

Retrieve the step event emitted at the end of an era, given a valid era identifier. The path URL is: <HOST:PORT>/step/<era-ID>. Enter a valid era identifier.

Example:

curl -s http://127.0.0.1:18888/step/7268

Missing Filter​

If no filter URL was specified after the root address (HOST:PORT), an error message will be returned.

Example:

curl http://127.0.0.1:18888
{"code":400,"message":"Invalid request path provided"}

Invalid Filter​

If an invalid filter was specified, an error message will be returned.

Example:

curl http://127.0.0.1:18888/other
{"code":400,"message":"Invalid request path provided"}
